# package-management Specification Delta

## REMOVED Requirements

### Requirement: 信息卡片展示

**原因**: 首页重构聚焦于 Web 管理核心功能，移除快速开始和安全提示信息卡片以简化界面。

**迁移**: 这些卡片的内容可以考虑在后续版本中整合到帮助文档或首次使用引导中。

#### Scenario: 显示快速开始信息卡片

**Given** 用户打开应用并访问首页
**When** 首页加载完成
**Then** 显示快速开始信息卡片
**And** 卡片包含标题"快速开始"
**And** 卡片包含描述文本"安装嵌入式 Web 服务包并开始管理您的本地 PCode 实例。"

#### Scenario: 显示安全提示信息卡片

**Given** 用户打开应用并访问首页
**When** 首页加载完成
**Then** 显示安全提示信息卡片
**And** 卡片包含标题"安全"
**And** 卡片包含描述文本"注重安全构建。您的数据保留在您的机器上。"

### Requirement: 设置提示文本

**原因**: 移除"更多设置即将推出"提示文本，清理冗余信息。

**迁移**: 无需迁移，该提示文本将被完全移除。

#### Scenario: 显示设置提示文本

**Given** 用户打开应用并访问首页
**When** 滚动到设置卡片区域
**Then** 在设置卡片底部显示提示文本"更多设置即将推出..."
**And** 提示文本使用灰色小字体

---

### Requirement: 首页依赖项检查功能

**原因**: 依赖项在版本安装后自动满足，无需在首页显示依赖项检查功能。

**迁移**: 如果后续需要检查依赖项，可以在版本管理页面中提供依赖检查入口。

#### Scenario: 显示依赖项检查卡片

**Given** 用户打开应用并访问首页
**When** 首页加载完成
**Then** 显示依赖项检查卡片
**And** 卡片包含标题"依赖项检查"
**And** 卡片包含检查和修复依赖项的按钮

#### Scenario: 依赖项状态检查

**Given** 依赖项检查卡片已显示
**When** 用户点击"检查依赖"按钮
**Then** 应用检查系统依赖项状态
**And** 显示依赖项检查结果（完整/缺失）
